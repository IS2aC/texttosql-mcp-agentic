DATABASE CONTEXT
---------------------------------------
This database manages employee leave information, tracking leave balances, requests, and types, while also storing user data, with foreign keys linking related tables to maintain data consistency and relationships.

---------------------------------------
DATABASE SCHEMA
---------------------------------------
## Database Schema

**core.leave_balances**
- id uuid NOT NULL
- user_id uuid NOT NULL (FK → core.users.id)
- leave_type_id uuid NOT NULL (FK → core.leave_types.id)
- year integer NOT NULL
- allocated_days numeric(5,2) NOT NULL
- used_days numeric(5,2) NOT NULL
- updated_at timestamp without time zone

**core.leave_requests**
- id uuid NOT NULL
- user_id uuid NOT NULL (FK → core.users.id)
- leave_type_id uuid NOT NULL (FK → core.leave_types.id)
- start_date date NOT NULL
- end_date date NOT NULL
- days_count numeric(5,2) NOT NULL
- status varchar(20) NOT NULL
- approved_by uuid (FK → core.users.id)
- approved_at timestamp without time zone
- created_at timestamp without time zone
- updated_at timestamp without time zone

**core.leave_types**
- id uuid NOT NULL
- name varchar(100) NOT NULL
- description text
- requires_approval boolean
- affects_balance boolean
- created_at timestamp without time zone

**core.users**
- id uuid NOT NULL
- email varchar(150) NOT NULL
- password_hash text NOT NULL
- first_name varchar(100) NOT NULL
- last_name varchar(100) NOT NULL
- manager_id uuid (FK → core.users.id)
- role varchar(20) NOT NULL
- status varchar(20)
- created_at timestamp without time zone
